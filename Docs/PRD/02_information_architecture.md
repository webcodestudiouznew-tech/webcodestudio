## Карта секций

| № | Секция | Тип секции | Layout pattern | Основная задача |
|---|---|---|---|---|
| 1 | Header | Header / Top Navigation | Sticky Header | Навигация и быстрый доступ к CTA |

| 3 | Trust Strip | Trust / Credibility Strip | Horizontal Card Strip | Быстрое доверие |

## CTA Strategy

- Не добавлять CTA в каждый блок по умолчанию.
- Использовать единый основной сценарий конверсии по странице: `Получить предложение`.
- Использовать единый вторичный сценарий: `Написать в Telegram`.
- Размещать CTA в тех точках, где после секции естественно возникает готовность к действию, а не просто завершился очередной блок.

### Распределение CTA по секциям

1. `Hero` — сильный primary CTA + secondary CTA.
2. `Для кого подходит` — без отдельного CTA.
3. `Почему WebCode` — обязателен мягкий inline CTA.
4. `Что получает бизнес` — повторный сильный CTA.
5. `Что входит в решение` — CTA опционален, только как спокойное продолжение оффера.
6. `Кейсы` — CTA обязателен.
7. `Тарифы` — CTA обязателен.
8. `Как проходит работа` — без обязательного большого CTA.
9. `FAQ` — CTA после снятия возражений желателен.
10. `Финальный CTA` — обязательный главный конверсионный экран.
11. `Footer` — utility CTA / контактные ссылки, не основной продающий экран.
