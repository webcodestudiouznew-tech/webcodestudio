# PRD Navigation

Эта папка нужна для быстрой навигации по PRD без постоянной загрузки целиком файла [WebCode_landind_PRD.md](../WebCode_landind_PRD.md).

## Канонический источник

- Главный источник истины: [WebCode_landind_PRD.md](../WebCode_landind_PRD.md)
- Документы в этой папке являются рабочими extracts для навигации и должны оставаться синхронизированными с master PRD.

## Состав

- [01_foundation.md](./01_foundation.md) — цель страницы, позиционирование, аудитория, tone of voice, CTA.
- [02_information_architecture.md](./02_information_architecture.md) — последовательность блоков, layout patterns, задачи секций.
- [03_sections_header_to_offer.md](./03_sections_header_to_offer.md) — детальные требования к `Header` → `Что входит в решение`.
- [04_sections_cases_to_footer.md](./04_sections_cases_to_footer.md) — детальные требования к `Кейсы` → `Footer`.
- [05_content_rules.md](./05_content_rules.md) — ограничения, требования к кейсам и финальный продуктовый итог.

## Как использовать

- Для продуктовых и контентных решений сначала смотреть сюда по нужному разделу.
- При любом спорном месте приоритет всегда у master PRD.
