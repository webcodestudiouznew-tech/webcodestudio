# accessibility_and_quality.md

Минимальный стандарт accessibility и UI quality для WebCode.

## 1. Accessibility Baseline
- Все интерактивные элементы должны быть доступны с клавиатуры.
- Все интерактивные элементы должны иметь видимый `focus` state.
- Контраст текста, кнопок и ссылок должен быть достаточным для уверенного чтения.
- Формы должны иметь labels, понятные ошибки и доступный success-state.
- Иконки без текстовой подписи должны иметь корректный доступный смысл или быть скрыты от screen readers.

## 2. Semantic Quality
- Заголовки должны соблюдать логичную иерархию.
- Кнопки и ссылки должны использоваться по назначению.
- Основные content blocks должны иметь понятную semantic structure.

## 3. Motion And Effects
- Визуальные эффекты не должны ухудшать читаемость.
- Анимации не должны мешать взаимодействию.
- Нужно учитывать `prefers-reduced-motion`, если в секции есть заметная motion-логика.

## 4. Performance-Aware UI
- Тяжёлые blur, shadow, layered gradients и animation stacks использовать осторожно.
- Декоративные решения не должны заметно ухудшать mobile rendering.
- Нельзя добавлять визуальную сложность без проверки её реальной ценности для интерфейса.

## 5. UI Definition Of Done
- UI-задача не считается завершённой, если секция ломается хотя бы в одной локали.
- UI-задача не считается завершённой, если mobile-версия визуально слабая или неудобная.
- UI-задача не считается завершённой, если в ней случайные размеры текста, отступов, padding или gap.
- UI-задача не считается завершённой, если CTA hierarchy неясна.
- UI-задача не считается завершённой, если появились новые визуальные паттерны без системной причины.

## 6. Final QA Checklist
- Проверить `RU / UZ / EN`.
- Проверить desktop и mobile.
- Проверить CTA, формы, focus states и feedback states.
- Проверить, что секция визуально соответствует уровню остальных секций страницы.
