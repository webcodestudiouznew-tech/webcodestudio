# Vercel Deploy Guide

## Для кого этот guide

Этот guide написан для первого deploy на `Vercel`, если вы раньше этого не делали.

Цель:
- подключить проект к `Vercel`
- выпустить сайт в production
- не сломать `SEO`, локали и форму заявки

## Что нужно заранее

Перед началом у вас должно быть:
- GitHub-репозиторий с проектом
- доступ к аккаунту `Vercel`
- доступ к DNS домена `webcode.uz`
- готовый production branch `main`
- проверенные env для формы и уведомлений

## Что важно понимать заранее

`Vercel` сам:
- собирает `Next.js` проект
- создаёт preview deploy для веток и pull request
- публикует production deploy из выбранной branch

Для этого проекта production должен идти из `main`.

## Шаг 1. Подготовить репозиторий

Перед подключением к `Vercel` убедитесь, что:
- все нужные изменения уже в Git
- production-ready код находится в `main`
- локально проходят:
  - `npm run lint`
  - `npm run build`

Если `main` ещё не обновлён, сначала закончите merge нужной ветки.

## Шаг 2. Зайти в Vercel и импортировать проект

1. Откройте `https://vercel.com`.
2. Войдите в аккаунт.
3. Нажмите `Add New...` -> `Project`.
4. Выберите GitHub-репозиторий `webcodestudio`.
5. Разрешите `Vercel` доступ к репозиторию, если он ещё не выдан.

## Шаг 3. Проверить import settings

На экране импорта проекта проверьте:
- `Framework Preset`: `Next.js`
- `Root Directory`: оставить пустым, если проект лежит в корне репозитория
- `Build Command`: оставить по умолчанию
- `Output Directory`: не менять
- `Install Command`: оставить по умолчанию

Если `Vercel` сам распознал `Next.js`, обычно ничего менять не нужно.

## Шаг 4. Добавить environment variables

До первого production deploy добавьте env в `Vercel`.

Откройте блок `Environment Variables` и добавьте:
- `NOTION_TOKEN`
- `NOTION_DATABASE_ID`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_GROUP_CHAT_ID`

Рекомендация:
- сначала добавить их хотя бы в `Production`
- если будете использовать preview для проверки формы, добавить и в `Preview`

Важно:
- значения вставлять без лишних пробелов
- после вставки ещё раз проверить имя каждой переменной
- не путать `TELEGRAM_GROUP_CHAT_ID` с username группы

## Шаг 5. Сделать первый deploy

После заполнения настроек:
1. Нажмите `Deploy`.
2. Дождитесь окончания сборки.
3. После успешного deploy откройте выданный URL от `Vercel`.

Это может быть ещё не production domain, а временный адрес вида:
- `project-name.vercel.app`

На этом этапе проверьте, что сайт вообще открывается.

## Шаг 6. Назначить production branch

После создания проекта:
1. Откройте проект в `Vercel`.
2. Перейдите в `Settings` -> `Git`.
3. Убедитесь, что `Production Branch` = `main`.

Это важно, потому что именно merge в `main` будет запускать production deploy.

## Шаг 7. Подключить домен

1. В проекте `Vercel` откройте `Settings` -> `Domains`.
2. Добавьте:
   - `webcode.uz`
3. При необходимости добавьте:
   - `www.webcode.uz`

Для этого проекта canonical host:
- `webcode.uz`

Если подключаете `www`, он должен вести на основной домен, а не жить отдельной версией сайта.

## Шаг 8. Настроить DNS

После добавления домена `Vercel` покажет, какие DNS-записи нужно добавить.

Обычно нужно:
- либо `A record`
- либо `CNAME`

Что делать:
1. Откройте панель управления доменом у регистратора или DNS-провайдера.
2. Добавьте записи ровно в том виде, как просит `Vercel`.
3. Сохраните изменения.
4. Вернитесь в `Vercel` и дождитесь, пока домен станет `Valid`.

Важно:
- DNS может примениться не мгновенно
- иногда это занимает несколько минут, иногда дольше

## Шаг 9. Проверить production deploy

Когда домен уже подключён:
1. Откройте `https://webcode.uz`.
2. Убедитесь, что сайт отвечает без ошибок.
3. Проверьте:
   - `/` редиректит на `/ru`
   - работают `/ru`, `/uz`, `/en`
   - открываются privacy pages
   - работает форма заявки

## Шаг 10. Что делать при следующих релизах

После первого deploy рабочая схема обычно такая:

1. Делаете изменения в feature branch.
2. Проверяете локально.
3. Получаете preview deploy от `Vercel`.
4. Проверяете preview.
5. Мержите в `main`.
6. `Vercel` автоматически запускает production deploy.
7. Проходите post-deploy проверки.

## Где в Vercel смотреть важные вещи

### Deployments

Здесь видно:
- все deploy
- статус сборки
- production и preview deploy
- логи build

### Settings -> Domains

Здесь видно:
- подключён ли домен
- валиден ли DNS
- какой домен production

### Settings -> Environment Variables

Здесь видно:
- какие env заданы
- для каких окружений они применяются

### Functions / Runtime logs

Здесь пригодится смотреть ошибки, если:
- не работает форма
- не уходит запись в `Notion`
- не уходит уведомление в `Telegram`

## Что делать, если deploy упал

### Если ошибка на build

Проверьте:
- проходит ли `npm run build` локально
- не забыты ли env
- нет ли синтаксической ошибки или проблем с импортами

### Если сайт открылся, но форма не работает

Проверьте:
- добавлены ли env в `Vercel`
- те ли значения вставлены
- есть ли ошибки в server logs

### Если домен не открывается

Проверьте:
- правильно ли добавлены DNS-записи
- действительно ли домен стал `Valid` в `Vercel`
- не остались ли старые конфликтующие записи

## Самая безопасная схема для первого релиза

1. Сначала подключить проект к `Vercel`.
2. Сначала проверить deploy на `*.vercel.app`.
3. Потом подключить домен.
4. Потом открыть live-домен и пройти post-deploy SEO checklist.

Так меньше риск, что вы одновременно разбираете и код, и DNS.

## Что не нажимать без необходимости

- не меняйте вручную `Build Command`, если нет причины
- не добавляйте кастомные redirects в `Vercel`, если это уже решено в коде
- не удаляйте старый working deployment до проверки нового
- не деплойте в production, если не проверен preview

## После успешного deploy

Сразу откройте:

- `https://webcode.uz/sitemap.xml`
- `https://webcode.uz/ru`
- `https://webcode.uz/uz`
- `https://webcode.uz/en`
