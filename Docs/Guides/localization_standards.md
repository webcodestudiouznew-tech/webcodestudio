# localization_standards.md

Стандарт качества локализации для `RU / UZ / EN`.

## 1. Общий принцип
- Локализация является частью UX, а не пост-обработкой готового интерфейса.
- Все новые тексты должны проектироваться сразу под `RU / UZ / EN`.
- Перевод не должен быть буквальным, если это ухудшает ясность, естественность или layout.

## 2. Rules For All Locales
- Смысл текста должен сохраняться, даже если структура фразы меняется.
- Нельзя смешивать стили обращения и тон внутри одной локали без причины.
- Терминология должна быть согласованной по всему сайту.
- UI-тексты должны быть короткими, ясными и пригодными для интерфейса, а не для длинного маркетингового текста.

## 3. Special Rule For UZ
- Все тексты для `UZ` должны звучать как естественный узбекский язык для пользователей в Узбекистане.
- Недопустимы буквальные кальки с русского, русизмы, смешение с русским и неестественные коммерческие формулировки.
- Если исходная русская фраза плохо переносится на узбекский, нужно адаптировать смысл, а не держаться за структуру оригинала.
- Приоритет у ясности, естественного порядка слов и принятого в Узбекистане коммерческого tone of voice.

## 4. Localization As Layout Constraint
- Каждая локаль должна проверяться как отдельный визуальный сценарий.
- Нельзя утверждать компонент только по `RU`, если `UZ` или `EN` ломают композицию.
- Если текст в локали слишком длинный, сначала адаптировать copy, затем typography и constraints, а не жертвовать читабельностью.
- Hero, bento-grid, pricing cards, form labels, buttons и footer требуют отдельной проверки для каждой локали.

## 5. Writing Rules For UI
- Кнопки, labels, badges и microcopy должны быть короткими и однозначными.
- Заголовки должны сохранять силу оффера, но не становиться громоздкими.
- Подписи, descriptions и helper text должны дополнять интерфейс, а не дублировать уже понятный смысл.

## 6. Final Check
- Проверить, что локализованный текст звучит как текст для реального пользователя, а не как машинный перевод.
- Проверить, что длина строк не ломает layout.
- Проверить, что tone of voice согласован между всеми секциями одной локали.
