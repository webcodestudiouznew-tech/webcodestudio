# conversion_standards.md

Стандарт реализации CTA, форм, messenger links и analytics touchpoints для лендинга.

`guide_landing_layouts_PRO.md` является главным документом для решения, какие конверсионные секции нужны на странице и где они должны находиться.
Этот документ отвечает за качество реализации уже выбранного conversion layer.

## 1. Общий принцип
- CTA, формы и messenger links должны быть реализованы как единая, ясная и низкофрикционная система контакта.
- Документ не определяет структуру лендинга и не заменяет guide по выбору секций.

## 2. CTA Hierarchy
- На странице должен быть понятный primary CTA.
- Secondary CTA должны поддерживать основной сценарий, а не конкурировать с ним.
- Messenger CTA допустимы как быстрый канал связи, если они не размывают основной путь конверсии.
- Тексты CTA должны быть конкретными и понятными; абстрактные формулировки без действия недопустимы.

## 3. Forms
- Форма должна содержать только поля, которые действительно нужны для первого контакта.
- Любое дополнительное поле должно иметь бизнес-обоснование.
- Обязательных полей должно быть минимально необходимое количество.
- Error states, pending state и success state обязательны.
- После отправки пользователь должен получить ясный feedback о результате.

## 4. Trust And Friction
- Любой текст рядом с формой должен снижать неопределённость и не добавлять лишний cognitive load.
- Если supporting block не помогает отправке формы, его нужно убрать или сократить.
- Нельзя перегружать финальную конверсионную секцию параллельными смыслами и декоративными блоками.

## 5. Messenger Links
- `Telegram`, `WhatsApp` и другие прямые каналы должны вести к реальному, рабочему контакту.
- Messenger CTA должны быть визуально согласованы с общей CTA hierarchy.
- Если messenger link используется как альтернатива форме, это должно быть явно понятно пользователю.

## 6. Analytics Touchpoints
- Все основные CTA и ключевые шаги формы должны быть готовы к аналитическим событиям.
- Минимум нужно отслеживать просмотр формы, попытку отправки, успешную отправку и клики по альтернативным каналам связи.
- Имена событий должны быть понятными и стабильными.

## 7. Conversion QA
- Проверить, что первичный сценарий действия очевиден уже при быстром просмотре страницы.
- Проверить, что CTA не теряются среди декоративных элементов.
- Проверить, что форма удобна и не создаёт лишнего трения.
- Проверить, что success-state объясняет следующий шаг или хотя бы подтверждает приём заявки.
