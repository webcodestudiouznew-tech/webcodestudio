# telegram_bot_group_setup.md

Документ описывает стандартный повторяемый процесс подключения автоматических уведомлений о новых заявках в `Telegram` через связку:
- `Telegram bot`
- рабочая `Telegram group`

Guide должен быть достаточным, чтобы этот же процесс можно было повторить на новом сайте без обращения к истории текущего проекта.

## 1. Что реализовано в проекте

Server action формы:
- [lead-form.ts](/mnt/e/DEV/webcodestudio/app/actions/lead-form.ts)

После успешной обработки заявки сервер:
- сохраняет лид в `Notion`
- пытается отправить уведомление в `Telegram`
- не ломает success-state формы, если `Telegram` временно недоступен

Используемые env-переменные:
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_GROUP_CHAT_ID`

Названия env зафиксированы в:
- [contact-links.ts](/mnt/e/DEV/webcodestudio/lib/contact-links.ts)

## 2. Короткая схема процесса

Для нового сайта порядок всегда такой:
1. создать `Telegram bot`
2. создать или выбрать рабочую `Telegram group`
3. добавить бота в группу
4. получить `TELEGRAM_GROUP_CHAT_ID`
5. добавить `TELEGRAM_BOT_TOKEN` и `TELEGRAM_GROUP_CHAT_ID` в env
6. реализовать server-side отправку через `Telegram Bot API`
7. встроить отправку в flow успешной обработки формы
8. проверить реальную доставку тестового сообщения и боевой заявки

## 3. Что нужно от пользователя

Для рабочего подключения нужны:
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_GROUP_CHAT_ID`
- подтверждение, что бот добавлен в группу
- подтверждение, что бот может отправлять сообщения в группу

## 4. Как создать бота

1. Открыть `@BotFather` в `Telegram`.
2. Отправить команду `/newbot`.
3. Задать имя и username бота.
4. Получить `bot token`.

Этот токен и будет значением `TELEGRAM_BOT_TOKEN`.

## 5. Как подготовить группу

1. Создать отдельную рабочую группу для заявок.
2. Добавить в неё созданного бота.
3. При необходимости выдать боту право на отправку сообщений.
4. Отправить в группу любое тестовое сообщение.

## 6. Как получить `TELEGRAM_GROUP_CHAT_ID`

### Способ через `getUpdates`

1. Добавить бота в группу.
2. Написать что-нибудь в группу.
3. Открыть:

```text
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/getUpdates
```

4. Найти в ответе поле `chat.id`.

Для группы `chat_id` обычно отрицательный, например:

```text
-1001234567890
```

Именно это значение и нужно использовать как `TELEGRAM_GROUP_CHAT_ID`.

## 7. Как подключить env в проект

Локально использовать:
- `.env.local`

Пример:

```env
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_GROUP_CHAT_ID=-1001234567890
```

Для production-like окружения те же переменные нужно добавить в env платформы деплоя.

## 8. Что нужно сделать в коде на новом сайте

Минимальный повторяемый implementation pattern:

1. Зафиксировать названия env-переменных в одном месте.
2. Реализовать server-side функцию отправки уведомления в `Telegram`.
3. Не отправлять уведомление с клиента.
4. Вызывать отправку только после успешной обработки основной заявки.
5. Сделать мягкий fallback:
   - если основной канал сохранения лида успешен, а `Telegram` недоступен, UI не должен падать
6. Логировать ошибку `Telegram` на сервере.

В текущем проекте это реализовано через:
- [contact-links.ts](/mnt/e/DEV/webcodestudio/lib/contact-links.ts)
- [lead-form.ts](/mnt/e/DEV/webcodestudio/app/actions/lead-form.ts)

## 9. Как работает уведомление

После отправки формы сервер собирает сообщение с такими полями:
- имя
- телефон
- `Telegram nickname`, если указан
- ниша бизнеса, если указана
- локаль формы
- время отправки

Сообщение отправляется через:

```text
https://api.telegram.org/bot<TELEGRAM_BOT_TOKEN>/sendMessage
```

## 10. Важная логика отказоустойчивости

Текущее поведение специально сделано мягким:
- если `Notion` сохранился, а `Telegram` недоступен, форма не должна падать для пользователя
- ошибка `Telegram` логируется на сервере
- success-state формы остаётся рабочим

## 11. Что проверять после подключения

После настройки нужно проверить:
- бот реально состоит в группе
- `chat_id` корректный
- после новой заявки сообщение приходит в группу
- в сообщении есть все нужные поля
- форма по-прежнему успешно пишет лид в `Notion`

## 12. Что делать при ошибке

Если сообщение не приходит:
- проверить `TELEGRAM_BOT_TOKEN`
- проверить `TELEGRAM_GROUP_CHAT_ID`
- проверить, что бот добавлен в группу
- проверить, что бот имеет право писать в группу
- проверить ответ `getUpdates`
- посмотреть server logs из `lead-form.ts`

## 13. Reusable Checklist для нового сайта

Перед завершением задачи на новом сайте проверить:
- бот создан
- группа создана или выбрана
- бот добавлен в группу
- `chat_id` подтверждён
- env подключены локально и в production-like окружении
- тестовое сообщение отправляется напрямую через `sendMessage`
- реальная заявка отправляет уведомление в группу
- ошибка `Telegram` не ломает success-state формы

## 14. Security Notes

- не хранить `TELEGRAM_BOT_TOKEN` в клиентском коде
- не коммитить реальные credentials в git
- если токен был где-либо раскрыт, сделать его ротацию через `@BotFather`
