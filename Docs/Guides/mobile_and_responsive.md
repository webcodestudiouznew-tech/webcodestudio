# mobile_and_responsive.md

Стандарт mobile-first и responsive-проверок для всех секций лендинга.

## 1. Базовый принцип
- Каждая секция должна сначала работать на mobile, затем масштабироваться на tablet и desktop.
- Desktop-композиция не является главным сценарием, если mobile-версия ломается или теряет смысл.
- Если композиция не переносится на mobile, нужно менять композицию, а не только уменьшать шрифт и padding.

## 2. Layout Behavior
- Нельзя допускать горизонтальный overflow, визуальное обрезание текста, неконтролируемые переносы и налезание элементов.
- Grid, bento, pricing cards, forms и CTA-группы должны проверяться отдельно на узких экранах.
- На mobile приоритет у читаемости, ясности CTA и правильного порядка информации.
- Допустимо скрывать второстепенные декоративные или supporting blocks на mobile, если они мешают конверсии или перегружают экран.

## 3. Text Resilience
- Заголовки должны выдерживать длинные строки в `RU / UZ / EN`.
- Если локаль ломает responsive layout, сначала адаптировать typography, width constraints и copy.
- Нельзя рассчитывать, что все языки займут одинаковое число строк.
- Для `UZ` и `EN` допускаются точечные правки `line-height`, `font-size`, `max-width` и переноса строк, если это улучшает UX.

## 4. Responsive Spacing
- Плотность mobile-интерфейса должна быть выше desktop, но без ощущения тесноты.
- Вертикальный ритм на mobile должен оставаться стабильным между секциями.
- Отступы внутри карточек и форм на mobile должны сохранять читаемость полей, заголовков и CTA.

## 5. CTA And Forms On Mobile
- Primary CTA не должен теряться ниже декоративного контента без причины.
- Кнопки должны иметь достаточную ширину и высоту для касания.
- Формы должны быть удобны для быстрого заполнения одной рукой.
- Supporting text рядом с формой на mobile нужно сокращать или убирать, если он мешает главному действию.

## 6. Responsive QA
- Проверить mobile viewport около `360px`.
- Проверить состояние около `390px`, `768px`, `1024px` и широкого desktop.
- Проверить поведение длинных заголовков, wide cards, multi-column blocks и footer.
- Проверить, что все локали выглядят управляемо на каждом ключевом брейкпоинте.
