# Design Direction

## Назначение документа

Этот файл фиксирует визуальное направление проекта WebCode на основе уже утверждённой `Hero`-секции.

Это не документ про:
- структуру лендинга;
- выбор секций;
- responsive-правила;
- accessibility;
- conversion logic;
- детальные implementation-стандарты.

За эти зоны отвечают другие guide-документы:
- `WebCode_landind_PRD.md` — контент, тексты, продуктовые смыслы и ограничения
- `Guides/guide_landing_layouts_PRO.md` — структура лендинга и логика секций
- `Guides/landing_ui_standards.md` — typography, spacing, visual execution
- `Guides/mobile_and_responsive.md` — mobile-first и responsive logic
- `Guides/localization_standards.md` — стандарты локализации
- `Guides/conversion_standards.md` — CTA, формы и conversion implementation
- `Guides/accessibility_and_quality.md` — accessibility и финальный quality baseline

Этот документ отвечает только за visual DNA проекта: настроение, характер, палитру, атмосферу, тип визуальных акцентов и общий образ интерфейса.

## Общий образ

WebCode должен выглядеть как современный, уверенный, прикладной digital-продукт для локального бизнеса в Узбекистане.

Нужное ощущение:
- надёжный
- собранный
- технологичный
- коммерчески убедительный
- визуально дорогой, но без fake premium
- современный, но не шаблонный

Чего нужно избегать:
- generic `SaaS`-эстетики без характера
- light corporate шаблонности
- noisy cyberpunk-подачи
- псевдо-люксовой графики без структуры

## Базовый визуальный язык

Основа проекта — тёмная тёплая среда с золотыми акцентами.

Ключевые признаки:
- глубокий тёмный фон
- мягкие золотые glow-облака
- светлая типографика
- акцентные золотые CTA и highlight-элементы
- ощущение продуктового интерфейса, а не абстрактной студийной страницы
- умеренное количество декоративных деталей
- сильная визуальная иерархия без перегруза

## Цветовое направление

### Основа

Базовая среда должна строиться вокруг тёплых тёмных тонов, близких к:
- `#252421`
- `#2d2b28`
- `#23221f`

Фон не должен быть плоским. Допустимы:
- мягкий vertical gradient
- radial glow
- локальные золотые засветы глубины

### Акцент

Главный акцент — тёплое золото.

Опорные оттенки:
- `#efcb65`
- `#d7b24c`
- `#d4af4a`
- `#f1cc6b`

Золото используется как акцент, а не как сплошная доминирующая заливка всей страницы.

### Текст и поверхности

- основной текст — white / near-white
- secondary text — белый с ослабленной плотностью
- borders и surfaces — тёплые, мягкие, не холодные и не графитово-серые
- поверхности не должны превращать весь интерфейс в набор одинаковых карточек

## Атмосфера и фон

Фон должен ощущаться как среда, а не как декоративная подложка.

Допустимо:
- radial glow
- мягкая глубина
- device-centric visual blocks
- редкие декоративные звёзды
- аккуратные тени

Нежелательно без сильной причины:
- шумные текстуры
- тяжёлые сетки на весь экран
- агрессивные abstract blobs
- большое количество случайной иконографики

## Типографический характер

Шрифтовая пара проекта:
- `Manrope` — заголовки и ключевые UI-акценты
- `Inter` — текст, навигация, supporting copy

Типографика должна ощущаться:
- плотной, но не тяжёлой
- уверенной, но не кричащей
- современной, но не модной ради модности

Основной характер:
- заголовки собранные и контролируемые
- body-text чистый и спокойный
- акцент на ясную иерархию, а не на эффектность ради эффектности

Детальные правила по typography находятся в `Guides/landing_ui_standards.md`.

## Визуальный язык компонентов

Проект должен собираться из компонентов, которые ощущаются частью одной продуктовой системы.

Опорные признаки:
- аккуратные радиусы
- мягкие тени
- умеренное использование borders
- тёплые surface-решения
- чистая линейная иконография
- CTA с чёткой визуальной иерархией

Важно:
- не вводить случайные новые стили карточек
- не менять резко характер кнопок, поверхностей и highlights от секции к секции
- не перегружать интерфейс декоративными контейнерами

## Hero как визуальный эталон

Утверждённая `Hero`-секция является главным визуальным референсом проекта.

Что из неё наследуется дальше:
- тёмно-золотая среда
- уровень контраста
- характер типографики
- стиль CTA
- визуальный баланс между контентом и product-feel
- общее ощущение premium utility, а не “красивой абстракции”

Новые секции не должны спорить с `Hero` по характеру.

## Motion Character

Motion в проекте должен ощущаться:
- мягким
- точным
- современным
- вторичным по отношению к смыслу

Допустимый характер motion:
- subtle reveal
- лёгкий hover-lift
- мягкое усиление яркости
- спокойные glow-эффекты

Недопустимый характер motion:
- резкие aggressive transitions
- тяжёлый continuous motion ради эффекта
- визуальный шум, который спорит с текстом и CTA

Детальные motion- и responsive-правила находятся в профильных guide-документах.

## Что нельзя делать

- уводить новые секции в светлый или холодный стиль без сильной причины
- смешивать с текущей системой случайные `blue` / `purple` акценты
- собирать интерфейс как generic template without identity
- добавлять случайные карточки, бордеры и glow-эффекты без функции
- делать визуал заметно слабее или дешевле по ощущению, чем утверждённая `Hero`

## Source Of Truth для visual direction

Если возникает спорное визуальное решение, ориентир такой:

1. Этот документ
2. Уже утверждённая `Hero`-секция
3. Актуальный visual language в коде проекта

Проверочный вопрос для любого нового решения:
- поддерживает ли оно тёмно-золотой product character
- не выглядит ли оно как чужой шаблон
- не разрушает ли ощущение цельного продукта
