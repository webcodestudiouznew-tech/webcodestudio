# SEO keyword brief для WebCode

## Цель
Сделать landing page более релевантным под реальные поисковые запросы в Узбекистане, не ломая текущий product positioning: сайт + CRM + Telegram/WhatsApp + домен + hosting + support.

## Главное правило
Не пытаться продвигать одну страницу сразу под все услуги.
Текущий landing page должен в первую очередь закрывать intent:
- создание сайта для бизнеса
- разработка сайта под ключ
- landing page / corporate website
- локально: Ташкент / Узбекистан

`CRM`, `Telegram/WhatsApp`, `3 языка`, `hosting`, `domain`, `support` — это не главный cluster, а differentiators и secondary keywords.

## Позиционирование страницы
Landing page должен ранжироваться не как “абстрактный digital product”, а как:
- услуга создания сайта
- услуга разработки landing page / business website
- услуга запуска сайта под ключ для бизнеса в Ташкенте и Узбекистане

Текущее positioning через `CRM`, `Telegram`, `WhatsApp`, `3 языка` оставить, но подать как преимущества внутри offer, а не как основную SEO тему страницы.

## Что нужно изменить

## 1. Обновить `Metadata.title` для всех локалей
### Задача
Переписать `title` так, чтобы в него вошел основной commercial query, но без keyword stuffing.

### Рекомендуемые варианты

#### RU
Основной вариант:
`Создание сайтов в Ташкенте для бизнеса | WebCode`

Альтернатива:
`Разработка сайтов в Ташкенте под ключ | WebCode`

#### UZ
Основной вариант:
`Toshkentda biznes uchun sayt yaratish | WebCode`

Альтернатива:
`Toshkentda kalit topshirish veb sayt yaratish | WebCode`

#### EN
Основной вариант:
`Website Development in Tashkent for Business | WebCode`

Альтернатива:
`Turnkey Website Development in Tashkent | WebCode`

### Правило
- `title` должен быть естественным.
- Не вставлять слишком много ключей сразу.
- Один page = один главный cluster.

---

## 2. Обновить `Metadata.description`
### Задача
Сделать `description` более search-oriented, но сохранить product differentiators.

### Рекомендуемые варианты

#### RU
`Создаем landing page и сайты для бизнеса в Ташкенте: CRM, Telegram/WhatsApp, domain, hosting и запуск под ключ в одном решении.`

#### UZ
`Toshkentdagi biznes uchun landing page va sayt yaratamiz: CRM, Telegram/WhatsApp, domain, hosting va tayyor ishga tushirish bitta yechimda.`

#### EN
`We build landing pages and business websites in Tashkent with CRM, Telegram/WhatsApp, domain, hosting and turnkey launch in one solution.`

### Правило
Description должен:
- включать главный search intent
- усиливать CTR
- не быть просто повтором `title`

## Дополнительные указания для Codex

### Не импровизировать сверх задачи
- Не придумывать новые product claims.
- Не добавлять новые секции без необходимости.
- Не переписывать тексты шире, чем нужно для SEO-задачи.

### Что считать источником истины
- Использовать текущую архитектуру проекта и существующий `AGENTS.md`.
- Использовать текущие message files как основной источник localized copy.
- Если есть выбор между “красивее” и “ближе к search intent”, выбирать вариант, который лучше закрывает search intent, но без keyword stuffing.

### Как принимать решения по copy
- Если в brief даны рекомендуемые варианты `title`, `description`, `H1`, использовать их как preferred option.
- Не изобретать полностью новые формулировки без необходимости.
- Если нужно отклониться от предложенного варианта, делать это только для:
  - естественности языка
  - длины строки
  - лучшей встраиваемости в UI

### Что обязательно показать в финальном ответе
- Какие именно `title` выбраны для `ru / uz / en`
- Какие именно `description` выбраны для `ru / uz / en`
- Какой именно `H1` выбран для `ru / uz / en`
- Какие `FAQ` были добавлены или изменены
- Какой route создан для `Privacy Policy`
- Какие public pages теперь входят в `sitemap`

### Что не делать
- Не менять visual layout.
- Не менять component structure без необходимости.
- Не добавлять dependencies без необходимости.
- Не трогать unrelated files.
