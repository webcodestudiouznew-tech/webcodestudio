# telegram_bot_group_lead_alerts.md

## Задача

Подключить автоматические уведомления о новых заявках формы WebCode в `Telegram` через связку:
- `Telegram bot`
- отдельная `Telegram group`

## Цель

После каждой успешной отправки формы заявка должна:
- сохраняться в `Notion`
- дублироваться как уведомление в рабочую `Telegram`-группу

## Почему выбран этот вариант

Связка `bot + group` подходит лучше, чем только личный чат:
- уведомления видит не один человек, а вся команда
- лиды отделены от личной переписки
- удобно обсуждать заявки прямо в группе
- решение легче масштабировать дальше

## Ожидаемое поведение

После успешной отправки формы сервер должен:
1. принять и провалидировать данные формы
2. сохранить лид в `Notion`
3. отправить уведомление в `Telegram`-группу
4. не ломать success-state формы, если уведомление в `Telegram` временно недоступно

## Данные, которые понадобятся от пользователя

- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_GROUP_CHAT_ID`
- подтверждение, что бот добавлен в группу
- подтверждение, что бот имеет право отправлять сообщения в группу

## Минимальный состав уведомления

Сообщение в `Telegram` должно содержать:
- имя
- телефон
- `Telegram nickname`, если указан
- нишу бизнеса, если указана
- локаль формы
- время отправки

## Технический scope

- добавить env-переменные для `Telegram bot`
- реализовать server-side отправку уведомления через `Telegram Bot API`
- встроить отправку в текущий flow `submitLeadForm`
- определить безопасное поведение при partial failure:
  - если `Notion` сохранился, а `Telegram` недоступен, форма не должна падать для пользователя
- при необходимости добавить guide-документ по настройке `Telegram bot` и получению `chat_id`

## Важные ограничения

- токен бота хранить только в env
- не отправлять уведомления с клиента
- не блокировать UI из-за вторичного канала уведомлений
- не менять состав формы без отдельного решения пользователя

## Definition Of Done

Задача считается выполненной, когда:
- бот подключён к группе
- сервер отправляет уведомление в группу после новой заявки
- уведомление содержит все нужные поля
- основная отправка формы остаётся стабильной
- ручная проверка подтверждает, что новая заявка дошла и в `Notion`, и в `Telegram`
